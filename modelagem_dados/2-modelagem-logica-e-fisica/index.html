    2. Modelagem lógica e física#

    1. Fundamentos e Funcionamento#

    • Definição técnica: A modelagem de dados é um processo usado para definir e analisar os requisitos de dados necessários para suportar os processos de negócio de uma organização. A modelagem lógica de dados refere-se à representação abstrata desses dados, focando na organização e na estrutura sem se preocupar com detalhes físicos de implementação. Já a modelagem física de dados lida com a conversão dessa representação abstrata em um esquema que pode ser implementado em um sistema de gerenciamento de banco de dados (SGBD).
    • Como funciona internamente: Na modelagem lógica, são utilizados diagramas de entidade-relacionamento (ER) para representar dados, suas relações e atributos. Na modelagem física, esses diagramas são transformados em tabelas, colunas, chaves primárias, chaves estrangeiras, índices, restrições e outros objetos de banco de dados.
    • Componentes principais: Entidades, Relacionamentos, Atributos (na modelagem lógica); Tabelas, Colunas, Chaves, Índices (na modelagem física).

    2. Estrutura Completa do Tema (Nível Prova)#

    • Regras:
      • A modelagem lógica deve preceder a física.
      • A normalização é aplicada durante a modelagem lógica para reduzir redundâncias.
    • Classificações:
      • Modelagem Conceitual: Representação de alto nível.
      • Modelagem Lógica: Foco na estrutura dos dados.
      • Modelagem Física: Foco na implementação no SGBD.
    • Categorias:
      • Entidades, Relacionamentos, Atributos (Lógica).
      • Tabelas, Views, Índices, Triggers (Física).
    • Exceções:
      • Denormalização pode ser aplicada na modelagem física por motivos de performance.

    3. Regras Técnicas e Comportamento#

    • Funcionamento detalhado: Na modelagem lógica, cada entidade é transformada em uma tabela na modelagem física. Relacionamentos lógicos são implementados por meio de chaves estrangeiras. Atributos tornam-se colunas das tabelas.
    • Limitações: A modelagem física está limitada pelas capacidades do SGBD escolhido.
    • Condições de uso: A modelagem lógica é independente de plataforma, enquanto a física deve considerar especificidades do SGBD (ex: MySQL, Oracle).

    4. Diferenças Críticas#

    • Comparar com conceitos semelhantes: A modelagem conceitual é muitas vezes confundida com a lógica, mas a conceitual é mais abstrata e focada na captura de requisitos de negócio, sem detalhar a estrutura dos dados.
    • Destacar o que a banca confunde: A banca pode confundir a necessidade de normalização na modelagem física, quando na verdade, a normalização é um conceito primariamente aplicado na modelagem lógica.

    5. Exemplos e Aplicações#

    • Exemplos técnicos:
      • Diagrama ER (Entidade-Relacionamento) para modelagem lógica.
      • Script SQL para criação de tabelas e índices na modelagem física.
    • Cenários reais: Transformação de um diagrama ER de um sistema de vendas online em um esquema de banco de dados, incluindo tabelas para clientes, produtos e pedidos.

    6. Pegadinhas de Prova#

    • Erros comuns:
      • Confundir normalização com denormalização.
      • Acreditar que a modelagem física não requer conhecimento sobre o SGBD.
    • Afirmações que parecem corretas:
      • “A modelagem lógica pode ser diretamente implementada em um SGBD.” (Errado, pois requer conversão para a modelagem física).
    • Termos que invalidam questões:
      • “Independência de dados é irrelevante na modelagem física.” (Errado, pois a independência de dados é crucial para a flexibilidade do design).

    7. Tabela de Fixação#

    ConceitoRegraPegadinha
    Modelagem LógicaFoco na estrutura dos dados, independente de SGBD.Confundir com modelagem conceitual.
    Modelagem FísicaImplementação específica do SGBD, inclui tabelas, índices, etc.Acreditar que não requer conhecimento do SGBD.
    NormalizaçãoAplicada na modelagem lógica para reduzir redundâncias.Confundir com denormalização.
    DenormalizaçãoPode ser aplicada na modelagem física para melhorar a performance.Pensar que é aplicada na modelagem lógica.

    8. Simulado (Certo ou Errado)#

    1. A modelagem lógica é independente de plataforma e SGBD. (CERTO)
    2. A normalização é um processo aplicado durante a modelagem física para melhorar a performance. (ERRADO)
    3. Tabelas, índices e chaves estrangeiras são componentes da modelagem lógica. (ERRADO)
    4. A modelagem física pode ser diretamente implementada sem a necessidade de uma modelagem lógica prévia. (ERRADO)
    5. Denormalização é um processo aplicado para melhorar a performance de leitura em bancos de dados. (CERTO)
    6. A modelagem conceitual detalha a estrutura dos dados e suas relações. (ERRADO)
    7. A independência de dados é um conceito irrelevante para a modelagem física. (ERRADO)
    8. Diagramas ER são utilizados na modelagem lógica para representar entidades e relacionamentos. (CERTO)

    Gabarito comentado:

    1. CERTO: A modelagem lógica é de fato independente de plataforma e SGBD, focando na estrutura dos dados e suas relações sem se preocupar com detalhes de implementação.
    2. ERRADO: A normalização é aplicada durante a modelagem lógica, não na física, e visa reduzir redundâncias e dependências.
    3. ERRADO: Tabelas, índices e chaves estrangeiras são componentes da modelagem física, que é a implementação específica no SGBD.
    4. ERRADO: A modelagem física requer uma modelagem lógica prévia para definir a estrutura dos dados de forma abstrata antes da implementação.
    5. CERTO: A denormalização é aplicada na modelagem física para melhorar a performance de leitura, especialmente em sistemas que realizam grandes volumes de consultas.
    6. ERRADO: A modelagem conceitual é uma representação de alto nível dos requisitos de negócio e não detalha a estrutura dos dados como a modelagem lógica.
    7. ERRADO: A independência de dados é crucial para a flexibilidade e manutenção do design do banco de dados, sendo relevante tanto na modelagem lógica quanto na física.
    8. CERTO: Diagramas ER são uma ferramenta fundamental na modelagem lógica para representar entidades, seus atributos e relacionamentos.
