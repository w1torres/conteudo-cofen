    1. Modelo entidade-relacionamento#

    1. Fundamentos e Funcionamento#

    • Definição técnica: O Modelo Entidade-Relacionamento (MER) é uma abordagem de modelagem de dados que visa representar as entidades relevantes para um domínio de negócio e os relacionamentos entre essas entidades. É amplamente utilizado na fase de modelagem conceitual de sistemas de banco de dados.
    • Como funciona internamente: O MER funciona como um diagrama que mapeia entidades, atributos e relacionamentos. Este modelo ajuda a visualizar a estrutura lógica de um banco de dados de forma abstrata, facilitando o entendimento das relações entre os dados antes da implementação física.
    • Componentes principais:
      • Entidades: Objetos ou conceitos que possuem dados armazenados. Exemplo: Cliente, Pedido.
      • Relacionamentos: Associações entre entidades. Exemplo: Um cliente faz um pedido.
      • Atributos: Propriedades ou características das entidades. Exemplo: Nome do Cliente, Data do Pedido.

    2. Estrutura Completa do Tema (Nível Prova)#

    • Regras:
      • Cada entidade deve ter um identificador único (chave primária).
      • Relacionamentos podem ser classificados como 1:1, 1:N, N:1, ou N:N.
    • Classificações:
      • Entidades: Fortes (independentes) e Fracas (dependentes de outra entidade para existir).
      • Relacionamentos: Identificadores (quando uma entidade fraca é identificada por sua relação com outra) e não identificadores.
    • Categorias:
      • Atributos: Simples, Compostos, Derivados, Multivalorados.
    • Exceções:
      • Atributos multivalorados devem ser tratados com cuidado na transição para o modelo lógico, podendo requerer a criação de entidades separadas.

    3. Regras Técnicas e Comportamento#

    • Funcionamento detalhado:
      • A chave primária de uma entidade deve ser única e não nula.
      • Relacionamentos N:N são frequentemente resolvidos com a criação de uma entidade associativa.
    • Limitações:
      • O MER é uma ferramenta de modelagem conceitual e não representa detalhes de implementação física.
    • Condições de uso:
      • Ideal para a fase inicial de design de sistemas de informação, onde o foco está na estruturação lógica dos dados.

    4. Diferenças Críticas#

    • Comparar com conceitos semelhantes:
      • Diferença entre MER e Modelo Relacional: O MER é usado para a modelagem conceitual, focando na estrutura e relações. O Modelo Relacional é uma implementação lógica, detalhando como os dados são armazenados em tabelas.
    • Destacar o que a banca confunde:
      • A banca pode confundir atributos compostos com entidades. É crucial entender que atributos compostos são apenas partes de uma entidade, não entidades separadas.

    5. Exemplos e Aplicações#

    • Exemplos técnicos:
      • Diagrama de Entidade-Relacionamento representando um sistema de biblioteca, onde Livro (Entidade) possui Título e Ano (Atributos) e está relacionado a Autor por um relacionamento Escreve.
    • Cenários reais:
      • Modelagem de um sistema de e-commerce, identificando entidades como Cliente, Produto, Pedido e seus respectivos atributos e relacionamentos.

    6. Pegadinhas de Prova#

    • Erros comuns:
      • Confundir entidades fracas com atributos multivalorados.
    • Afirmações que parecem corretas:
      • “Atributos derivados são armazenados no banco de dados.” (Falso, pois são calculados a partir de outros atributos).
    • Termos que invalidam questões:
      • Uso incorreto da terminologia de relacionamentos (ex: dizer que um relacionamento 1:N é um atributo).

    7. Tabela de Fixação#

    ConceitoRegraPegadinha
    Entidades FortesIndependentes, possuem chave primária própria.Confundir com entidades fracas.
    Relacionamentos N:NResolvidos com entidade associativa.Pensar que podem existir sem entidade associativa.
    Atributos DerivadosCalculados a partir de outros atributos, não armazenados diretamente.Acreditar que são armazenados no banco de dados.

    8. Simulado (Certo ou Errado)#

    1. (C) Cada entidade no modelo entidade-relacionamento deve ter um identificador único.
    2. (E) Atributos derivados são armazenados no banco de dados junto com os demais atributos da entidade.
    3. (C) Relacionamentos N:N entre entidades são geralmente implementados por meio de uma entidade associativa.
    4. (E) No modelo entidade-relacionamento, entidades fracas podem existir sem entidades fortes.
    5. (C) Atributos compostos podem ser divididos em subpartes, que representam propriedades mais específicas da entidade.
    6. (E) Um relacionamento 1:1 sempre indica que ambas as entidades envolvidas são entidades fracas.
    7. (C) No modelo entidade-relacionamento, relacionamentos podem ter atributos próprios.
    8. (E) Atributos multivalorados são representados no modelo entidade-relacionamento como entidades separadas.

    Gabarito comentado:

    1. Correto. Identificadores únicos são fundamentais para diferenciar cada instância de uma entidade.
    2. Errado. Atributos derivados são calculados e não armazenados diretamente.
    3. Correto. Entidades associativas são usadas para resolver relacionamentos N:N, criando uma nova entidade que referencia as chaves primárias das entidades originais.
    4. Errado. Entidades fracas dependem de entidades fortes para sua existência, não podendo existir sozinhas.
    5. Correto. Atributos compostos permitem a decomposição em partes menores, facilitando a organização dos dados.
    6. Errado. Relacionamentos 1:1 não implicam automaticamente que ambas as entidades são fracas; a classificação depende de outros fatores.
    7. Correto. Relacionamentos podem possuir atributos que descrevem características da associação entre entidades.
    8. Errado. Atributos multivalorados não são representados como entidades separadas; eles podem exigir tratamento especial na modelagem, mas não se tornam entidades.
