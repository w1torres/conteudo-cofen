    3. Data Warehouse e Data Lake#

    1. Fundamentos e Funcionamento#

    Definição técnica#

    • Data Warehouse (DW): É uma coleção centralizada de dados históricos de diferentes fontes, organizada para suportar decisões de negócios. Utiliza um esquema dimensional ou normalizado para armazenar dados.
    • Data Lake: Armazena grandes volumes de dados brutos em seu formato nativo até que sejam necessários. Diferentemente do DW, não é estruturado nem processado até o momento de sua utilização.

    Como funciona internamente#

    • Data Warehouse: Funciona com a extração de dados de sistemas transacionais (OLTP), sua transformação (limpeza, agregação, e enriquecimento) e carga no DW (processo ETL). Os dados são armazenados em tabelas que são frequentemente desnormalizadas para otimizar a leitura.
    • Data Lake: Armazena dados de diversas fontes sem alterá-los. Quando os dados são necessários, são extraídos, transformados e carregados (processo ELT) para análise. Suporta análises de big data e machine learning.

    Componentes principais#

    • Data Warehouse: Modelo de dados, armazenamento de dados, motor de consulta, ferramentas de ETL, front-end de relatórios.
    • Data Lake: Armazenamento de objetos ou blobs, serviços de ingestão de dados, serviços de processamento e análise, segurança e governança de dados.

    2. Estrutura Completa do Tema (Nível Prova)#

    Listar TODOS os elementos relevantes:#

    • Regras:
      • DW: Utiliza esquemas estrela, floco de neve ou normalizado para modelagem.
      • Data Lake: Não impõe esquema até o momento da leitura (schema-on-read).
    • Classificações:
      • DW: Analítico, operacional.
      • Data Lake: Baseado em arquivos, objeto, ou bloco.
    • Categorias:
      • DW: Histórico, integrado, não volátil, temporal.
      • Data Lake: Raw (bruto), Processed (processado), Refined (refinado).
    • Exceções:
      • DW: Não é ideal para dados não estruturados.
      • Data Lake: Pode tornar-se um “Data Swamp” se mal gerenciado.

    3. Regras Técnicas e Comportamento#

    Funcionamento detalhado#

    • DW: Otimizado para carga de trabalho de leitura intensiva com dados estruturados. Ideal para relatórios e análises históricas.
    • Data Lake: Otimizado para armazenar grandes volumes de dados brutos. Ideal para exploração de dados, machine learning e análises ad-hoc.

    Limitações#

    • DW: Menos flexível para dados não estruturados e big data.
    • Data Lake: Requer mais governança para evitar tornar-se um “Data Swamp”.

    Condições de uso#

    • DW: Quando a análise de negócios precisa de dados históricos estruturados e confiáveis.
    • Data Lake: Quando há necessidade de armazenar grandes volumes de dados brutos para análises futuras.

    4. Diferenças Críticas#

    • Estruturação de Dados: DW requer estruturação prévia dos dados; Data Lake armazena dados em seu formato bruto.
    • Processamento de Dados: DW utiliza ETL; Data Lake utiliza ELT.
    • Tipo de Dados: DW é ideal para dados estruturados; Data Lake suporta tanto dados estruturados quanto não estruturados.

    5. Exemplos e Aplicações#

    • DW: Análise de vendas históricas para prever tendências futuras.
    • Data Lake: Armazenamento de logs de web para análise de comportamento do usuário com machine learning.

    6. Pegadinhas de Prova#

    • Erros comuns:
      • Confundir o processo ETL com ELT.
      • Acreditar que DW é adequado para todos os tipos de dados.
    • Afirmações que parecem corretas:
      • “Data Lake substitui a necessidade de um Data Warehouse.” (Falso, eles servem propósitos diferentes.)
    • Termos que invalidam questões:
      • “Data Lake é ideal para relatórios estruturados de negócios.” (Falso, esse é o papel do DW.)

    7. Tabela de Fixação#

    Data LakeArmazena dados brutos em seu formato nativo.“Substitui a necessidade de um DW”
    ETL vs. ELTDW utiliza ETL; Data Lake utiliza ELT.“ETL é usado para Data Lake”

    8. Simulado (Certo ou Errado)#

    1. Data Lakes são otimizados para armazenar grandes volumes de dados estruturados. (ERRADO)
    2. Data Warehouses utilizam o processo ELT para transformação de dados. (ERRADO)
    3. Um Data Lake pode armazenar dados não estruturados, como imagens e vídeos. (CERTO)
    4. Data Warehouses são ideais para análises preditivas e machine learning. (ERRADO)
    5. Data Lakes requerem esquemas definidos no momento da ingestão de dados. (ERRADO)
    6. A modelagem de dados em um Data Warehouse é geralmente desnormalizada para otimizar a leitura. (CERTO)
    7. Data Lakes podem se tornar “Data Swamps” se não forem bem gerenciados. (CERTO)
    8. O processo ETL é característico de Data Lakes. (ERRADO)

    Gabarito comentado:#

    1. Data Lakes são otimizados para armazenar grandes volumes de dados, incluindo dados não estruturados, não apenas estruturados.
    2. Data Warehouses tradicionalmente utilizam o processo ETL (Extract, Transform, Load), não ELT.
    3. Correto. Data Lakes são projetados para armazenar todos os tipos de dados, incluindo não estruturados.
    4. Data Warehouses são otimizados para relatórios e análises históricas, não necessariamente para análises preditivas e machine learning.
    5. Data Lakes utilizam o conceito de schema-on-read, o que significa que o esquema é aplicado no momento da leitura, não na ingestão.
    6. A desnormalização em Data Warehouses é uma prática comum para otimizar consultas de leitura.
    7. Sem governança adequada, Data Lakes podem acumular dados de baixa qualidade, tornando-se “Data Swamps”.
    8. O processo ELT é mais associado a Data Lakes, permitindo maior flexibilidade no processamento de dados.
