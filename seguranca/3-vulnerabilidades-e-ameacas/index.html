    3. Vulnerabilidades e ameaças#

    1. Fundamentos e Funcionamento#

    Vulnerabilidades são falhas ou fraquezas em um sistema que podem ser exploradas por ameaças para causar danos, perda ou acesso não autorizado a informações. As ameaças, por outro lado, são potenciais causas de incidentes que podem resultar em dano para um sistema ou organização. Elas podem ser intencionais, como ataques cibernéticos, ou não intencionais, como falhas de software ou desastres naturais.

    2. Estrutura Completa do Tema (Nível Prova)#

    • Tipos de Ataques: Phishing, Man-in-the-Middle (MitM), Denial of Service (DoS), Distributed Denial of Service (DDoS), Ransomware, SQL Injection, Cross-Site Scripting (XSS), Malware.
    • Classificações de Vulnerabilidades: Segurança de rede, segurança de aplicação, segurança operacional, vulnerabilidades físicas.
    • Categorias de Ameaças: Internas vs. Externas, Ativas vs. Passivas, Intencionais vs. Não intencionais.
    • Mecanismos de Defesa: Firewalls, Sistemas de Detecção de Intrusão (IDS), Sistemas de Prevenção de Intrusão (IPS), Antivírus, Criptografia, Autenticação Multifator, Políticas de Segurança.

    3. Regras Técnicas e Comportamento#

    • Firewalls funcionam como uma barreira entre redes confiáveis e não confiáveis, filtrando o tráfego com base em regras predefinidas.
    • IDS monitora o tráfego de rede para atividades suspeitas e gera alertas.
    • IPS é semelhante ao IDS, mas tem a capacidade de bloquear tentativas de intrusão detectadas automaticamente.
    • Criptografia protege informações transformando-as em um formato ilegível sem a chave correta.
    • Autenticação Multifator requer duas ou mais credenciais independentes para verificar a identidade do usuário, aumentando a segurança.

    4. Diferenças Críticas#

    • DoS vs. DDoS: DoS é um ataque de um único ponto de origem que visa tornar um recurso indisponível. DDoS, por outro lado, utiliza múltiplos sistemas comprometidos para realizar o ataque.
    • Phishing vs. Spear Phishing: Phishing é uma técnica de engenharia social genérica, enquanto Spear Phishing é altamente personalizado para o alvo.

    5. Exemplos e Aplicações#

    • SQL Injection: Um ataque que explora vulnerabilidades de validação de entrada para executar comandos SQL maliciosos.
    • Ransomware: Malware que criptografa arquivos do usuário, exigindo pagamento para descriptografá-los.

    6. Pegadinhas de Prova#

    • Afirmar que IDS e IPS são a mesma coisa.
    • Confundir criptografia simétrica com assimétrica.
    • Acreditar que todas as vulnerabilidades são exploráveis remotamente.

    7. Tabela de Fixação#

    ConceitoRegraPegadinha
    PhishingAtaque de engenharia social para obter informações confidenciais.Confundir com Spear Phishing.
    CriptografiaUsa chaves para cifrar e decifrar dados.Achar que não pode ser quebrada.

    8. Simulado (Certo ou Errado)#

    1. Um firewall pode efetivamente bloquear todos os tipos de ataques cibernéticos. (ERRADO)
    2. A criptografia assimétrica utiliza a mesma chave para cifrar e decifrar a informação. (ERRADO)
    3. Phishing é uma forma de ataque que sempre requer a interação do usuário. (CERTO)
    4. Um Sistema de Detecção de Intrusão (IDS) pode bloquear automaticamente um ataque. (ERRADO)
    5. SQL Injection é um ataque que se aproveita de vulnerabilidades em aplicações web. (CERTO)
    6. Ransomware é um tipo de malware que impede o acesso a sistemas até que um resgate seja pago. (CERTO)
    7. A autenticação de dois fatores elimina completamente o risco de ataques de phishing. (ERRADO)
    8. Spear Phishing é menos direcionado do que o Phishing tradicional. (ERRADO)

    Gabarito comentado:#

    1. Errado. Firewalls são eficazes contra muitos tipos de ataques, mas não todos. Ataques sofisticados ou direcionados internamente podem contorná-los.
    2. Errado. A criptografia assimétrica usa uma chave pública para cifrar e uma chave privada para decifrar.
    3. Certo. Phishing geralmente envolve enganar o usuário para que ele forneça informações confidenciais.
    4. Errado. IDS gera alertas para atividades suspeitas, mas não bloqueia ataques. Sistemas de Prevenção de Intrusão (IPS) têm essa capacidade.
    5. Certo. SQL Injection explora vulnerabilidades de validação de entrada em aplicações web para executar comandos SQL maliciosos.
    6. Certo. Ransomware bloqueia ou criptografa arquivos, exigindo pagamento para recuperação.
    7. Errado. Embora a autenticação de dois fatores aumente a segurança, não elimina completamente o risco de ataques de phishing, especialmente os mais sofisticados.
    8. Errado. Spear Phishing é altamente direcionado, ao contrário do Phishing tradicional, que é mais genérico.
