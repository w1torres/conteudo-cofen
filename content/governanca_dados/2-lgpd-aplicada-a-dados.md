# 2. LGPD aplicada a dados

## 1. Fundamentos e Funcionamento

A Lei Geral de Proteção de Dados (LGPD), Lei nº 13.709/2018, estabelece regras sobre o tratamento de dados pessoais, incluindo os digitais, por parte de organizações públicas e privadas. Seu objetivo é proteger os direitos fundamentais de liberdade e de privacidade e o livre desenvolvimento da personalidade da pessoa natural.

### Definição técnica
Tratamento de dados pessoais refere-se a qualquer operação realizada com dados pessoais, como a coleta, produção, recepção, classificação, utilização, acesso, reprodução, transmissão, distribuição, processamento, arquivamento, armazenamento, eliminação, avaliação ou controle da informação, modificação, comunicação, transferência, difusão ou extração.

### Como funciona internamente
A LGPD aplica-se a qualquer operação de tratamento de dados pessoais, exceto para fins pessoais, jornalísticos, artísticos, acadêmicos, e para a segurança pública, defesa nacional, segurança do Estado e atividades de investigação e repressão de infrações penais.

### Componentes principais
- Dados pessoais: Informação relacionada a pessoa natural identificada ou identificável.
- Dados sensíveis: Dados pessoais sobre origem racial ou étnica, convicção religiosa, opinião política, filiação a sindicato ou a organização de caráter religioso, filosófico ou político, dado referente à saúde ou à vida sexual, dado genético ou biométrico.

## 2. Estrutura Completa do Tema (Nível Prova)

### Regras
- Consentimento: O tratamento de dados deve ser feito com o consentimento do titular dos dados.
- Finalidade: Os dados só podem ser coletados para finalidades legítimas, específicas, explícitas e informadas ao titular.
- Necessidade: Limitação do tratamento ao mínimo necessário para realização de suas finalidades.

### Classificações
- Titular dos dados: Pessoa natural a quem se referem os dados pessoais.
- Controlador: Pessoa física ou jurídica a quem competem as decisões referentes ao tratamento de dados pessoais.
- Operador: Pessoa física ou jurídica que realiza o tratamento de dados pessoais em nome do controlador.

### Categorias
- Dados pessoais
- Dados pessoais sensíveis
- Dados anonimizados

### Exceções
- Tratamento para fins pessoais, jornalísticos, artísticos ou acadêmicos
- Segurança pública, defesa nacional, segurança do Estado e investigação de infrações penais

## 3. Regras Técnicas e Comportamento

### Funcionamento detalhado
O tratamento de dados deve seguir os princípios da finalidade, adequação, necessidade, livre acesso, qualidade dos dados, transparência, segurança, prevenção, não discriminação, responsabilização e prestação de contas.

### Limitações
Não é permitido o tratamento de dados pessoais para fins discriminatórios ilícitos ou abusivos.

### Condições de uso
O consentimento deve ser fornecido de forma livre, informada e inequívoca, podendo ser revogado a qualquer momento pelo titular dos dados.

## 4. Diferenças Críticas

### Comparar com conceitos semelhantes
- **LGPD vs GDPR**: Embora ambas as leis compartilhem princípios semelhantes de proteção de dados, a LGPD é a legislação brasileira, enquanto o GDPR (Regulamento Geral sobre a Proteção de Dados) é a legislação europeia. Existem diferenças nas penalidades e no escopo de aplicação.

### Destacar o que a banca confunde
- **Dados pessoais sensíveis vs Dados anonimizados**: Dados pessoais sensíveis requerem proteção especial e consentimento específico para seu tratamento, enquanto dados anonimizados, que não podem ser associados a um indivíduo, não são sujeitos à LGPD.

## 5. Exemplos e Aplicações

- **Cenário de consentimento**: Uma empresa que deseja enviar newsletters para seus clientes precisa obter o consentimento explícito dos mesmos para coletar e usar seus endereços de e-mail para esse fim.

## 6. Pegadinhas de Prova

- **Erros comuns**: Afirmar que a LGPD se aplica a qualquer tratamento de dados pessoais, sem considerar as exceções.
- **Afirmações que parecem corretas**: "Dados anonimizados são considerados dados pessoais sob a LGPD." (Incorreto, pois dados anonimizados não são considerados dados pessoais se a anonimização for irreversível).
- **Termos que invalidam questões**: Uso do termo "dados públicos" sem o devido contexto, pois dados pessoais disponibilizados publicamente pelo titular ainda são protegidos pela LGPD.

## 7. Tabela de Fixação

| Conceito             | Regra                                                                 | Pegadinha                              |
|----------------------|-----------------------------------------------------------------------|----------------------------------------|
| Consentimento        | Deve ser livre, informado e inequívoco.                               | Consentimento tácito é suficiente.     |
| Dados sensíveis      | Requerem consentimento específico e proteção especial.                | São tratados da mesma forma que outros dados pessoais. |
| Anonimização         | Dados anonimizados não são considerados dados pessoais.               | Dados anonimizados ainda são regulados pela LGPD. |

## 8. Simulado (Certo ou Errado)

1. ( ) A LGPD aplica-se exclusivamente a dados pessoais digitais.
2. ( ) O consentimento para o tratamento de dados pessoais pode ser revogado a qualquer momento pelo titular.
3. ( ) Dados anonimizados, sob a LGPD, são considerados dados pessoais se a anonimização for reversível.
4. ( ) O tratamento de dados pessoais para fins de segurança pública não é coberto pela LGPD.
5. ( ) Dados pessoais sensíveis podem ser tratados sem consentimento do titular em casos específicos previstos em lei.
6. ( ) A LGPD permite o tratamento de dados pessoais sem consentimento para o cumprimento de obrigação legal pelo controlador.
7. ( ) A anonimização de dados pessoais é uma forma de tratamento de dados que remove a possibilidade de associação do dado ao seu titular.
8. ( ) Todos os dados coletados por organizações públicas e privadas estão sujeitos às regras da LGPD, sem exceções.

### Gabarito comentado:

1. Errado. A LGPD aplica-se tanto a dados digitais quanto não digitais.
2. Certo. O consentimento é uma das bases legais para o tratamento de dados e pode ser revogado a qualquer momento.
3. Certo. Dados anonimizados que podem ser reversíveis não perdem a característica de dados pessoais.
4. Errado. A LGPD possui exceções para o tratamento de dados para fins de segurança pública, defesa nacional, segurança do Estado e investigação de infrações penais.
5. Certo. Existem previsões legais que permitem o tratamento de dados sensíveis sem consentimento, como para proteção da vida do titular.
6. Certo. A LGPD prevê o tratamento de dados sem consentimento para cumprimento de obrigação legal ou regulatória pelo controlador.
7. Certo. A anonimização, quando irreversível, faz com que os dados tratados não possam ser associados a um indivíduo, deixando de ser considerados dados pessoais.
8. Errado. Existem exceções na LGPD para o tratamento de dados pessoais, como para fins pessoais, jornalísticos, artísticos ou acadêmicos.
