# 1. Princípios de governança de dados

## 1. Definição e Funcionamento

- **Conceito direto**: A governança de dados é um conjunto de processos que assegura que os dados importantes para a organização sejam formalmente gerenciados ao longo de todo o seu ciclo de vida. Isso inclui a qualidade, a segurança, a disponibilidade e a integridade dos dados.
- **Como funciona**: Envolve a definição de políticas, procedimentos, padrões e métricas que garantem o uso eficaz e eficiente dos dados, permitindo que a organização atinja seus objetivos.
- **Onde é aplicado**: Em todos os setores onde os dados são um recurso crítico, incluindo saúde, finanças, governo, e tecnologia da informação.

## 2. Pontos Críticos de Prova

- **Regras importantes**: A Lei Geral de Proteção de Dados (LGPD) e a Lei 14.133/2021 são fundamentais no contexto brasileiro de governança de dados.
- **Classificações**: Dados podem ser classificados como estruturados, semi-estruturados e não estruturados. A governança deve abranger todos esses tipos.
- **Exceções**: Nem todos os dados necessitam do mesmo nível de governança. Dados críticos para a operação ou conformidade exigem atenção especial.

## 3. Pegadinhas Comuns

- **Confusões frequentes**: Confundir governança de dados com administração de dados. A governança foca na gestão e política, enquanto a administração lida com aspectos técnicos.
- **Termos que mudam o sentido**: Acreditar que "dados protegidos pela LGPD" se referem apenas a dados pessoais, quando na verdade a lei abrange qualquer informação relacionada a pessoa natural identificada ou identificável.
- **Diferenças entre conceitos próximos**: Governança de dados versus gestão de dados. A governança define as políticas e padrões, enquanto a gestão implementa e opera dentro desses parâmetros.

## 4. Aplicação Prática

- **Exemplo simples**: Implementação de uma política de classificação de dados que define quais dados são considerados sensíveis e requerem criptografia.
- **Situação real**: Uma empresa de e-commerce aplicando a LGPD ao mapear todos os dados pessoais coletados, definindo a base legal para cada tipo de dado e implementando medidas de segurança adequadas.

## 5. Tabela Resumo

| Conceito | Definição | Pegadinha |
|----------|-----------|-----------|
| Governança de Dados | Conjunto de processos para gerir o ciclo de vida dos dados. | Confundir com administração de dados. |
| LGPD | Lei que regula o tratamento de dados pessoais. | Pensar que se aplica apenas a dados sensíveis. |
| Classificação de Dados | Processo de identificar o nível de sensibilidade dos dados. | Ignorar a necessidade de proteger dados não estruturados. |

## 6. Simulado (Certo ou Errado)

1. A governança de dados é responsável apenas pela segurança dos dados. (Errado)
2. A LGPD aplica-se exclusivamente a dados pessoais sensíveis. (Errado)
3. Todos os dados dentro de uma organização devem ser criptografados. (Errado)
4. A classificação de dados é um componente essencial da governança de dados. (Certo)
5. Governança de dados e gestão de dados são termos intercambiáveis. (Errado)

**Gabarito comentado**:

1. **Errado**: A governança de dados abrange mais do que apenas a segurança, incluindo qualidade, disponibilidade e integridade dos dados.
2. **Errado**: A LGPD aplica-se a qualquer dado pessoal, não apenas os sensíveis.
3. **Errado**: Nem todos os dados necessitam de criptografia; a necessidade depende da classificação de sensibilidade dos dados.
4. **Certo**: A classificação de dados é fundamental para determinar o nível de proteção e políticas aplicáveis.
5. **Errado**: Governança de dados foca nas políticas e padrões, enquanto a gestão de dados lida com a implementação operacional dessas políticas.
