    2. Python#

    1. Fundamentos e Funcionamento#

    Definição técnica: Python é uma linguagem de programação de alto nível, interpretada, multiparadigma (procedural, orientada a objetos e funcional), com tipagem dinâmica e forte tipagem. É amplamente usada em automação, scripts, web, ciência de dados e administração de sistemas.

    Como funciona internamente:

    • Na implementação mais comum (CPython), o código-fonte .py é compilado para bytecode.
    • Esse bytecode pode ser armazenado em arquivos .pyc no diretório __pycache__.
    • O bytecode é executado pela Python Virtual Machine (PVM).
    • Em CPython, há o GIL (Global Interpreter Lock), que limita a execução simultânea de múltiplas threads Python no mesmo processo para CPU-bound.

    Componentes principais:

    • Interpretador
    • Runtime
    • Biblioteca padrão
    • Gerenciador de pacotes: pip
    • Ambientes virtuais: venv
    • Módulos e pacotes
    • REPL (modo interativo)

    2. Estrutura Completa do Tema (Nível Prova)#

    Checklist de prova#

    • Python é interpretada, mas em CPython há compilação prévia para bytecode.
    • Tipagem é dinâmica: tipo associado ao objeto, não à variável.
    • Tipagem é forte: Python evita coerções implícitas perigosas.
    • Blocos são definidos por indentação, não por chaves.
    • Variáveis não precisam de declaração prévia.
    • Tudo em Python é objeto (em sentido prático de prova).
    • Estruturas básicas: int, float, bool, str, list, tuple, set, dict.
    • Mutáveis: list, dict, set.
    • Imutáveis: int, float, bool, str, tuple.
    • list mantém ordem e aceita duplicatas.
    • set não aceita duplicatas e não garante indexação.
    • dict mapeia chave→valor; nas versões atuais preserva ordem de inserção.
    • tuple é imutável.
    • Controle de fluxo: if, for, while, match (versões recentes).
    • Tratamento de exceções: try, except, else, finally, raise.
    • Funções são definidas com def; funções anônimas com lambda.
    • Classes são definidas com class; há herança e polimorfismo.
    • Módulos podem ser importados com import e from ... import ....
    • Automação comum: arquivos, diretórios, processos, rede, scraping, APIs.
    • Bibliotecas frequentes em automação: os, sys, pathlib, shutil, subprocess, requests (externa).

    3. Regras Técnicas e Comportamento#

    Variáveis e tipos#

    x = 10
x = "texto"

    A mesma variável pode referenciar objetos de tipos diferentes em momentos distintos.

    Mutabilidade#

    lista = [1, 2]
lista.append(3)   # altera o objeto

tupla = (1, 2)
# tupla.append(3) -> erro

    Questão clássica: imutabilidade da tupla não impede que ela contenha objetos mutáveis.

    Passagem de argumentos#

    Python usa modelo de passagem por atribuição de objeto (object reference / call by sharing). Em prova, cuidado com “passagem por valor” ou “por referência” em sentido estrito.

    Escopo#

    • Local
    • Enclosing
    • Global
    • Built-in

    Palavras-chave:

    • global: altera variável global
    • nonlocal: altera variável do escopo imediatamente externo

    Laços#

    for i in range(3):
    print(i)   # 0,1,2

    range(3) não gera 1 a 3, mas 0 a 2.

    Exceções#

    try:
    x = 1 / 0
except ZeroDivisionError:
    print("erro")
finally:
    print("sempre executa")

    finally executa mesmo havendo exceção, salvo interrupções anormais severas.

    Compreensões#

    quadrados = [x*x for x in range(5)]

    São sintaxe concisa para criação de coleções.

    Automação#

    Principais frentes:

    • Arquivos: leitura/escrita
    • Sistema operacional: manipular diretórios, variáveis de ambiente
    • Processos: executar comandos externos
    • Web: consumir APIs, scraping
    • Agendamento: scripts periódicos via SO ou bibliotecas

    Exemplo:

    from pathlib import Path

pasta = Path("relatorios")
pasta.mkdir(exist_ok=True)

with open(pasta / "saida.txt", "w", encoding="utf-8") as f:
    f.write("Automação em Python")

    Limitações relevantes#

    • CPython com GIL pode prejudicar paralelismo real em tarefas CPU-bound com threads.
    • Python tende a ser mais lento que linguagens compiladas como C/C++.
    • Dependências externas exigem gestão de ambiente.

    4. Diferenças Críticas#

    Python x Java/C#

    • Python: tipagem dinâmica, sintaxe enxuta, sem compilação nativa obrigatória.
    • Java/C#: tipagem estática.
    • C/C++: compiladas para código nativo; Python geralmente executa via interpretador/VM.

    Lista x Tupla#

    • Lista: mutável
    • Tupla: imutável
      Pegadinha: ambas podem ser iteradas e armazenar elementos heterogêneos.

    Set x Dict#

    • set: conjunto de valores únicos
    • dict: pares chave-valor

    = x ==#

    • = atribui
    • == compara igualdade

    is x ==#

    • ==: igualdade de valor
    • is: identidade do objeto
      Banca gosta de trocar uma pela outra.

    Thread x Processo em Python#

    • Threads compartilham memória do processo.
    • Processos têm memória separada.
    • Em CPython, para CPU-bound, processos costumam ser mais eficazes que threads.

    5. Exemplos e Aplicações#

    Automação de arquivos#

    import os

for nome in os.listdir("."):
    if nome.endswith(".log"):
        print("Arquivo de log:", nome)

    Execução de comando externo#

    import subprocess

resultado = subprocess.run(["echo", "teste"], capture_output=True, text=True)
print(resultado.stdout)

    Consumo de API#

    import requests

resp = requests.get("https://api.github.com")
print(resp.status_code)

    requests não faz parte da biblioteca padrão.

    Script de automação simples#

    from pathlib import Path
import shutil

origem = Path("entrada")
destino = Path("backup")
destino.mkdir(exist_ok=True)

for arq in origem.glob("*.txt"):
    shutil.copy(arq, destino / arq.name)

    6. Pegadinhas de Prova#

    • “Python é puramente interpretada.”
      Errado em sentido técnico estrito: em CPython há compilação para bytecode antes da execução.

    • “Python é fracamente tipada.”
      Errado. É dinamicamente tipada, porém fortemente tipada.

    • “Tuplas não podem conter objetos mutáveis.”
      Errado. A tupla é imutável; seus elementos podem ser mutáveis.

    • “Set é indexado como lista.”
      Errado.

    • “Dict não mantém ordem.”
      Em provas atuais, cuidado: nas versões modernas, mantém ordem de inserção.

    • “finally só executa se não houver exceção.”
      Errado.

    • “requests pertence à biblioteca padrão.”
      Errado.

    • Termos absolutos invalidadores: “sempre”, “nunca”, “exclusivamente”, “obrigatoriamente”.

    7. Tabela de Fixação#

    ConceitoRegraPegadinha
    TipagemDinâmica e forteConfundir dinâmica com fraca
    ExecuçãoCPython compila para bytecode e executa na PVMDizer que não há compilação
    IndentaçãoDelimita blocosAfirmar uso de chaves
    ListaMutável, ordenadaDizer que é imutável
    TuplaImutávelDizer que não pode conter mutáveis
    SetNão duplicado, sem indexaçãoTratar como lista
    DictChave-valor; preserva ordem de inserção nas versões atuaisDizer que é desordenado em qualquer versão
    isIdentidadeConfundir com igualdade
    finallyTende a executar sempreDizer que depende de exceção
    GILRestringe paralelismo de threads CPU-bound em CPythonDizer que impede toda concorrência

    8. Simulado (Certo ou Errado)#

    1. Python é uma linguagem de tipagem estática, mas com sintaxe flexível para automação.
    2. Em CPython, o código-fonte pode ser convertido em bytecode antes da execução.
    3. Objetos do tipo list são imutáveis, o que favorece segurança em automação.
    4. O operador is verifica se dois objetos possuem o mesmo valor lógico.
    5. O bloco finally em tratamento de exceções é destinado à execução de rotinas de limpeza.
    6. A biblioteca subprocess pode ser usada para executar comandos do sistema operacional.
    7. Em Python, conjuntos (set) admitem elementos duplicados, desde que sejam do mesmo tipo.
    8. Em CPython, o GIL pode limitar o ganho de desempenho de múltiplas threads em tarefas intensivas de CPU.

    Gabarito comentado#

    1. Errado.
    Pegadinha: Python não é de tipagem estática; é dinâmica.

    2. Certo.
    Em CPython, há compilação para bytecode, depois execução pela VM.

    3. Errado.
    list é mutável. Imutável é tuple.

    4. Errado.
    is verifica identidade, não igualdade de valor. Igualdade é ==.

    5. Certo.
    finally é clássico para fechamento de arquivo, conexão e limpeza.

    6. Certo.
    subprocess é biblioteca padrão para execução de processos externos.

    7. Errado.
    set não admite duplicados, independentemente do tipo.

    8. Certo.
    Pegadinha: o GIL não impede toda concorrência, mas limita threads em CPU-bound no CPython.
