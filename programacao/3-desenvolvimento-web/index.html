    3. Desenvolvimento Web#

    1. Fundamentos e Funcionamento#

    O desenvolvimento Web envolve a criação de aplicações que operam na Internet ou em uma rede de intranet. Essas aplicações são acessadas por usuários através de navegadores web e são construídas usando uma combinação de linguagens de programação e tecnologias, incluindo HTML5, CSS, JavaScript, XML e JSON. O desenvolvimento Web pode ser dividido em front-end (cliente-side) e back-end (server-side), com algumas aplicações utilizando ambas as abordagens para oferecer uma experiência completa ao usuário.

    • Definição técnica: O desenvolvimento Web é o processo de programação ou codificação que habilita a funcionalidade de um website ou aplicação web, conforme os requisitos do proprietário.

    • Como funciona internamente: O navegador do usuário solicita uma página web ao servidor através do protocolo HTTP/HTTPS. O servidor, então, responde com os arquivos necessários (HTML, CSS, JavaScript, etc.). O navegador interpreta esses arquivos e exibe a página para o usuário. Durante esse processo, pode haver comunicação adicional com o servidor para buscar dados dinâmicos, geralmente através de AJAX, utilizando XML ou JSON.

    • Componentes principais: HTML5 fornece a estrutura básica da página, CSS é usado para controle de layout e design, enquanto JavaScript permite interatividade. XML e JSON são formatos usados para troca de dados entre cliente e servidor.

    2. Estrutura Completa do Tema (Nível Prova)#

    • HTML5:

      • Elementos semânticos (<article>, <section>, <nav>, <footer>, etc.)
      • Formulários e tipos de entrada (<input type="email">, <input type="date">, etc.)
      • Novos atributos (placeholder, required, pattern)
      • API de vídeo e áudio, Gráficos Canvas, SVG
      • Armazenamento local e sessão

    • CSS:

      • Seletores e pseudo-classes
      • Box model, Flexbox, Grid
      • Media queries para design responsivo
      • Transições e animações
      • Variáveis CSS

    • JavaScript:

      • Sintaxe e estruturas de controle
      • Manipulação do DOM
      • Eventos
      • AJAX e Fetch API
      • Promises e Async/Await

    • XML:

      • Estrutura baseada em árvore
      • Atributos e elementos
      • Namespaces
      • DTD e Schema para validação

    • JSON:

      • Notação de objetos JavaScript
      • Arrays e objetos
      • Serialização e desserialização
      • Uso com AJAX e APIs RESTful

    3. Regras Técnicas e Comportamento#

    • HTML5: Deve ser usado doctype <!DOCTYPE html> para especificar HTML5. Elementos semânticos melhoram a acessibilidade e são preferidos sobre o uso indiscriminado de <div>s.

    • CSS: A cascata e herança são princípios fundamentais. O uso de Flexbox ou Grid depende do layout desejado; Flexbox é melhor para componentes de uma dimensão, enquanto Grid é superior para layouts bidimensionais.

    • JavaScript: Deve-se evitar o uso de var em favor de let e const para declaração de variáveis, proporcionando escopo de bloco. As funções assíncronas facilitam o trabalho com operações que dependem de resposta, como buscar dados de uma API.

    • XML vs. JSON: XML é mais verboso e suporta namespaces, o que é útil em documentos complexos. JSON é mais leve, fácil de ler e escrever, e é a escolha preferida para APIs web e configurações.

    4. Diferenças Críticas#

    • HTML5 vs. HTML Tradicional: HTML5 introduz elementos semânticos e APIs para funcionalidades modernas da web, como armazenamento local e manipulação de mídia, não presentes no HTML tradicional.

    • CSS Grid vs. Flexbox: Grid é para layouts bidimensionais (linhas e colunas), enquanto Flexbox é para layouts unidimensionais (ou linha ou coluna).

    • JavaScript vs. jQuery: JavaScript é a linguagem de programação usada para criar interações na web, enquanto jQuery é uma biblioteca JavaScript que simplifica a manipulação do DOM, eventos e animações. jQuery é menos usado hoje em dia, com muitas de suas conveniências agora incorporadas em JavaScript puro.

    • XML vs. JSON: JSON é geralmente mais rápido e mais fácil de usar com JavaScript, enquanto XML é melhor para documentos com estruturas complexas e precisa de validação.

    6. Pegadinhas de Prova#

    • Confundir elementos semânticos do HTML5 com simples <div>s.
    • Usar Grid para layouts unidimensionais, onde Flexbox seria mais adequado.
    • Confundir a sintaxe de JSON com a de um objeto JavaScript.
    • Acreditar que jQuery é necessário para todas as interações JavaScript na web.

    8. Simulado (Certo ou Errado)#

    1. HTML5 introduziu elementos semânticos que não existiam no HTML4. (Certo)
    2. CSS Grid é mais adequado para layouts unidimensionais do que Flexbox. (Errado)
    3. JavaScript não pode manipular o DOM sem a ajuda de bibliotecas como jQuery. (Errado)
    4. JSON é uma forma de serializar objetos JavaScript que não suporta arrays. (Errado)
    5. XML é preferencialmente usado em comparação ao JSON para APIs web devido à sua verbosidade. (Errado)
    6. O uso de <nav> em HTML5 é restrito a links de navegação interna de uma página. (Errado)
    7. Promises em JavaScript são usadas para lidar com operações assíncronas, substituindo a necessidade de callbacks aninhados. (Certo)
    8. CSS Variables permitem a reutilização de valores em diferentes regras, facilitando a manutenção do código. (Certo)

    Gabarito comentado:

    1. Certo. HTML5 introduziu elementos como <article>, <section>, <nav>, e <footer>, que ajudam na semântica e acessibilidade da web.
    2. Errado. CSS Flexbox é projetado para layouts unidimensionais, enquanto CSS Grid é a melhor escolha para layouts bidimensionais.
    3. Errado. JavaScript puro (vanilla JS) é perfeitamente capaz de manipular o DOM sem a necessidade de bibliotecas externas.
    4. Errado. JSON suporta tanto objetos quanto arrays, sendo uma forma eficiente de representar dados estruturados.
    5. Errado. Devido à sua leveza e facilidade de uso com JavaScript, JSON é geralmente preferido sobre XML para APIs web.
    6. Errado. O elemento <nav> é destinado a marcar a navegação principal de um site, mas não se limita a links internos.
    7. Certo. Promises são uma abordagem moderna para lidar com assincronicidade em JavaScript, evitando o “callback hell”.
    8. Certo. Variáveis CSS permitem que os desenvolvedores definam valores que podem ser reutilizados em todo o documento CSS, tornando o código mais fácil de manter.
