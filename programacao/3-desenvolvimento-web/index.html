    3. Desenvolvimento Web#

    1. Fundamentos e Funcionamento#

    Desenvolvimento Web envolve a construção de aplicações e páginas acessadas via navegador, baseadas principalmente em HTML5 (estrutura/semântica), CSS (apresentação), JavaScript (comportamento), além de formatos de intercâmbio de dados como XML e JSON.

    Definição técnica#

    • HTML5: linguagem de marcação para estruturar conteúdo.
    • CSS: linguagem de folhas de estilo para definir layout, cores, tipografia e responsividade.
    • JavaScript: linguagem de programação interpretada, orientada a eventos, usada no navegador e também no servidor.
    • XML: linguagem de marcação extensível, hierárquica, voltada à representação e troca de dados.
    • JSON: formato textual leve para troca de dados, baseado em pares chave-valor e listas.

    Como funciona internamente#

    1. O navegador requisita um recurso ao servidor via HTTP/HTTPS.
    2. Recebe HTML, CSS, JS, imagens etc.
    3. O HTML é parseado e gera a DOM.
    4. O CSS é parseado e gera a CSSOM.
    5. DOM + CSSOM compõem a árvore de renderização.
    6. O JavaScript pode manipular DOM/CSSOM e alterar a interface.
    7. XML/JSON costumam trafegar como dados entre cliente e servidor.

    Componentes principais#

    • Cliente: navegador.
    • Servidor Web: entrega recursos.
    • DOM: representação em árvore do documento HTML/XML.
    • Eventos: clique, submit, load, input etc.
    • APIs Web: fetch, localStorage, geolocation, canvas.

    2. Estrutura Completa do Tema (Nível Prova)#

    Checklist de HTML5#

    • Estrutura básica: <!DOCTYPE html>, <html>, <head>, <body>
    • Elementos semânticos: <header>, <nav>, <main>, <section>, <article>, <aside>, <footer>
    • Elementos multimídia: <audio>, <video>, <source>, <track>
    • Formulários: input, type, required, placeholder, pattern, autocomplete
    • Atributos globais: id, class, style, title, hidden, data-*
    • Semântica ≠ apresentação
    • HTML5 não é linguagem de programação

    Checklist de CSS#

    • Sintaxe: seletor + declarações
    • Cascata, herança e especificidade
    • Seletores: elemento, classe, id, atributo, pseudo-classe, pseudo-elemento
    • Box model: content, padding, border, margin
    • display: block, inline, inline-block, none, flex, grid
    • Posicionamento: static, relative, absolute, fixed, sticky
    • Responsividade: media queries
    • id tem maior especificidade que class

    Checklist de JavaScript#

    • Tipagem dinâmica
    • var, let, const
    • Escopo global, de função e de bloco
    • Funções, objetos, arrays
    • Eventos e manipulação do DOM
    • Assincronismo: callback, Promise, async/await
    • JS no navegador é single-threaded com event loop
    • === compara valor e tipo; == faz coerção

    Checklist de XML#

    • Extensível e hierárquico
    • Sensível a maiúsculas/minúsculas
    • Deve ser bem-formado
    • Pode ser válido contra DTD/XSD
    • Suporta namespaces
    • Tags de abertura e fechamento obrigatórias, salvo forma vazia <tag/>

    Checklist de JSON#

    • Formato leve de dados
    • Objetos {} e arrays []
    • Chaves devem estar entre aspas duplas
    • Valores: string, número, boolean, null, objeto, array
    • Não aceita comentários no padrão JSON
    • Mais leve e simples que XML em geral

    3. Regras Técnicas e Comportamento#

    HTML5#

    • <!DOCTYPE html> informa ao navegador o uso do padrão HTML5.
    • Elementos semânticos melhoram acessibilidade e SEO, mas não alteram automaticamente o visual.
    • Atributos required, pattern, min, max realizam validação no cliente, mas não substituem validação no servidor.
    • id deve ser único no documento; class pode se repetir.

    CSS#

    • Cascata: múltiplas regras podem incidir sobre o mesmo elemento.
    • Especificidade: inline > #id > .classe/atributos/pseudo-classes > elemento/pseudo-elemento.
    • Herança: nem toda propriedade é herdada.
    • display:none remove o elemento do fluxo e da renderização; visibility:hidden oculta, mas preserva espaço.
    • position:absolute posiciona em relação ao ancestral posicionado mais próximo.

    JavaScript#

    • var tem escopo de função e sofre hoisting; let e const têm escopo de bloco.
    • const impede reatribuição da variável, mas não torna imutável o objeto referenciado.
    • JavaScript é interpretado/JIT e orientado a protótipos.
    • Eventos podem ser tratados inline, por propriedade (onclick) ou por addEventListener.
    • JSON.parse() converte texto JSON em objeto; JSON.stringify() faz o inverso.

    XML#

    • Documento bem-formado exige:
      • um único elemento raiz;
      • tags corretamente aninhadas;
      • fechamento correto;
      • atributos com aspas.
    • Validação por DTD ou XSD verifica conformidade estrutural.
    • XML é autodescritivo, porém mais verboso.

    JSON#

    • Não suporta tags de fechamento.
    • Não admite vírgula final no padrão estrito.
    • É independente de linguagem, embora muito associado ao JavaScript.
    • Em APIs REST, JSON é o formato mais comum.

    4. Diferenças Críticas#

    HTML x XML#

    • HTML: vocabulário predefinido, foco em apresentação/estrutura de páginas.
    • XML: vocabulário extensível, foco em dados.
    • HTML tolera certas omissões; XML é rigoroso.

    XML x JSON#

    • XML usa tags; JSON usa pares chave-valor.
    • XML suporta namespaces e esquemas mais robustos.
    • JSON tende a ser mais compacto e mais simples para APIs.
    • Questão clássica: JSON não substitui integralmente XML em todos os cenários.

    id x class#

    • id: único.
    • class: reutilizável.
    • Em CSS, #id possui maior especificidade que .class.

    == x ===#

    • ==: igualdade com coerção.
    • ===: igualdade estrita.
    • Banca gosta de afirmar que são equivalentes: errado.

    display:none x visibility:hidden#

    • display:none: não ocupa espaço.
    • visibility:hidden: ocupa espaço, mas não aparece.

    5. Exemplos e Aplicações#

    HTML5 + CSS + JS#

    <!DOCTYPE html>
<html lang="pt-BR">
<head>
  <meta charset="UTF-8">
  <style>
    .destaque { color: blue; }
    #titulo { font-weight: bold; }
  </style>
</head>
<body>
  <main>
    <h1 id="titulo" class="destaque">Página</h1>
    <input type="text" id="nome" required>
    <button id="btn">Enviar</button>
  </main>

  <script>
    document.getElementById("btn").addEventListener("click", function() {
      const nome = document.getElementById("nome").value;
      alert("Olá, " + nome);
    });
  </script>
</body>
</html>

    XML#

    <servidor>
  <nome>web01</nome>
  <porta>8080</porta>
</servidor>

    JSON#

    {
  "nome": "web01",
  "porta": 8080
}

    Cenário real#

    • Front-end envia formulário.
    • JavaScript intercepta evento e usa fetch.
    • Servidor responde em JSON.
    • Interface é atualizada sem recarregar toda a página.

    6. Pegadinhas de Prova#

    • “HTML5 é linguagem de programação” → Errado.
    • “CSS é usado para estruturar semanticamente o conteúdo” → Errado.
    • “JSON aceita comentários” → Errado no padrão.
    • “XML exige esquema XSD para existir” → Errado; XSD é opcional.
    • “const torna objetos imutáveis” → Errado.
    • “display:none e visibility:hidden têm o mesmo efeito” → Errado.
    • “id pode ser repetido se estiver em seções diferentes” → Errado no mesmo documento.
    • “JavaScript é compilado exclusivamente antes da execução” → Errado como regra geral de prova.

    Termos que invalidam:

    • “sempre”, “nunca”, “exclusivamente”, “obrigatoriamente”, “apenas”.

    CSSDefine apresentação visualNão substitui HTML
    JavaScriptManipula DOM, eventos e lógica== não é igual a ===
    XMLExtensível, hierárquico e rigorosoNão é mais simples que JSON
    JSONLeve, chave-valor, muito usado em APIsNão aceita comentários no padrão
    idDeve ser únicoTem maior especificidade que classe
    classPode se repetirMenor especificidade que id
    display:noneRemove da renderização e do fluxoDiferente de visibility:hidden

    8. Simulado (Certo ou Errado)#

    1. O HTML5 é uma linguagem de programação voltada à construção da lógica de páginas web.
    2. Em CSS, um seletor por id possui, em regra, maior especificidade que um seletor por classe.
    3. Em JavaScript, o operador === realiza comparação sem coerção de tipos.
    4. Um documento XML bem-formado pode existir sem estar validado por XSD ou DTD.
    5. JSON permite comentários de linha e de bloco, assim como JavaScript.
    6. A propriedade display:none oculta o elemento, mas mantém o espaço originalmente ocupado no layout.
    7. O atributo required em formulários HTML5 elimina a necessidade de validação no servidor.
    8. XML é, por definição, menos verboso que JSON.

    Gabarito comentado#

    1. Errado. HTML5 é linguagem de marcação. Pegadinha: confundir estrutura com lógica.
    2. Certo. #id > .classe em especificidade.
    3. Certo. === compara valor e tipo, sem coerção.
    4. Certo. Bem-formado é diferente de válido. Validação é opcional.
    5. Errado. JSON padrão não aceita comentários.
    6. Errado. Quem mantém espaço é visibility:hidden.
    7. Errado. Validação no cliente não substitui validação no servidor.
    8. Errado. Em geral, XML é mais verboso que JSON.
