    2. LGPD aplicada a dados#

    1. Fundamentos e Funcionamento#

    Definição técnica#

    A Lei Geral de Proteção de Dados (LGPD), Lei nº 13.709/2018, estabelece regras sobre o tratamento de dados pessoais, incluindo os digitais, por parte de organizações públicas e privadas, com o objetivo de proteger a liberdade e a privacidade das pessoas naturais.

    Como funciona internamente#

    A LGPD determina que o tratamento de dados deve seguir princípios como finalidade, adequação, necessidade, entre outros, garantindo que os dados pessoais sejam tratados de forma transparente e segura, e apenas para os propósitos legítimos e explícitos aos quais foram coletados.

    Componentes principais#

    • Dados pessoais e dados sensíveis
    • Titular dos dados
    • Controlador e operador
    • Autoridade Nacional de Proteção de Dados (ANPD)

    2. Estrutura Completa do Tema (Nível Prova)#

    Regras#

    • Consentimento do titular para tratamento de dados
    • Direitos do titular (acesso, correção, exclusão, etc.)
    • Obrigações do controlador e operador

    Classificações#

    • Dados pessoais: Informações relacionadas à pessoa natural identificada ou identificável.
    • Dados sensíveis: Dados sobre origem racial ou étnica, convicção religiosa, opinião política, saúde ou vida sexual, dados genéticos ou biométricos.

    Categorias#

    • Tratamento de dados
    • Transferência internacional de dados

    Exceções#

    • Tratamento de dados para fins de segurança pública, defesa nacional, segurança do Estado, ou investigações de infrações penais.

    3. Regras Técnicas e Comportamento#

    Funcionamento detalhado#

    O tratamento de dados deve ser documentado e justificado legalmente através de uma das bases legais, como consentimento, cumprimento de obrigação legal, proteção da vida, entre outros.

    Limitações#

    • Não é permitido o tratamento de dados sensíveis sem o consentimento específico e destacado do titular, exceto em casos previstos por lei.
    • A transferência internacional de dados só é permitida para países ou organismos internacionais que proporcionem grau de proteção de dados pessoais adequado ao previsto na LGPD.

    Condições de uso#

    • Necessidade de nomeação de um Encarregado de Proteção de Dados (DPO).
    • Necessidade de relatório de impacto à proteção de dados pessoais para operações de tratamento de risco.

    4. Diferenças Críticas#

    Comparar com conceitos semelhantes#

    • LGPD vs GDPR: Embora ambas legislações compartilhem muitos princípios, a LGPD é a legislação brasileira, enquanto o GDPR é o regulamento europeu. Existem diferenças nas penalidades e na abrangência de aplicação.

    Destacar o que a banca confunde#

    • Dados pessoais vs Dados sensíveis: A banca pode confundir ao tratar ambos como se não houvesse distinção significativa nas regras de tratamento.

    5. Exemplos e Aplicações#

    • Cenário de consentimento: Uma empresa que deseja enviar newsletters para seus clientes precisa obter o consentimento explícito para o tratamento dos dados pessoais para esse fim específico.

    6. Pegadinhas de Prova#

    • Erros comuns: Afirmar que todos os tratamentos de dados necessitam de consentimento do titular.
    • Afirmações que parecem corretas: “Dados anonimizados são considerados dados pessoais sob a LGPD.” (Dados anonimizados, desde que não possam ser revertidos, não são considerados dados pessoais.)
    • Termos que invalidam questões: Uso da expressão “dados públicos” sem a devida contextualização, pois mesmo dados públicos podem estar sujeitos à LGPD dependendo do contexto de tratamento.

    7. Tabela de Fixação#

    ConceitoRegraPegadinha
    Dados PessoaisInformações relacionadas à pessoa natural identificada ou identificável.Confundi-los com dados anonimizados.
    ConsentimentoNecessário para tratamento de dados, exceto em casos previstos por lei.Achar que é sempre obrigatório.
    Dados SensíveisTratamento restrito e necessita consentimento específico.Tratá-los como dados pessoais comuns.

    8. Simulado (Certo ou Errado)#

    1. ( ) A LGPD aplica-se exclusivamente a dados digitais.
    2. ( ) Dados anonimizados, que não podem ser revertidos, são considerados dados pessoais pela LGPD.
    3. ( ) O consentimento é a única base legal para o tratamento de dados pessoais.
    4. ( ) Dados sensíveis podem ser tratados sem consentimento do titular em determinadas condições previstas em lei.
    5. ( ) A transferência internacional de dados é livre, independentemente do país de destino.
    6. ( ) É obrigatório o tratamento de dados de acordo com a finalidade que foi informada ao titular.
    7. ( ) Todos os tratamentos de dados pessoais devem ser previamente comunicados à ANPD.
    8. ( ) A LGPD estabelece a necessidade de nomeação de um Encarregado de Proteção de Dados (DPO) para todas as organizações.

    Gabarito comentado:#

    1. Errado. A LGPD aplica-se tanto a dados digitais quanto não digitais.
    2. Errado. Dados anonimizados, desde que irreversíveis, não são considerados dados pessoais.
    3. Errado. Existem diversas bases legais para o tratamento de dados pessoais além do consentimento.
    4. Certo. Existem condições específicas em que os dados sensíveis podem ser tratados sem consentimento.
    5. Errado. A transferência internacional de dados só é permitida para países que proporcionem um grau de proteção adequado ou mediante garantias específicas.
    6. Certo. O tratamento de dados deve respeitar a finalidade para a qual os dados foram coletados.
    7. Errado. Nem todos os tratamentos de dados pessoais precisam ser comunicados à ANPD, mas devem seguir as diretrizes da LGPD.
    8. Errado. A nomeação de um DPO é obrigatória para algumas organizações, dependendo da natureza e do volume de dados tratados.
