    1. Virtualização#

    1. Fundamentos e Funcionamento#

    Virtualização é uma tecnologia que permite criar e gerenciar múltiplas instâncias de recursos, como sistemas operacionais, servidores, ou aplicações, em um único hardware físico. Essas instâncias são conhecidas como máquinas virtuais (VMs). A virtualização é realizada por meio de um software chamado hypervisor, que se situa entre o hardware e as VMs, gerenciando a distribuição dos recursos físicos para as VMs de maneira eficiente e isolada.

    Como funciona internamente#

    O hypervisor intercepta as chamadas ao hardware feitas pelas VMs, gerenciando essas solicitações de modo que cada VM tenha acesso aos recursos de que precisa, quando precisa, sem interferir nas demais VMs. Isso é feito através de técnicas como partição de tempo (time slicing) para CPUs e alocação dinâmica de memória.

    Componentes principais#

    • Hypervisor: Software central da virtualização, responsável por criar e gerenciar as VMs.
    • Máquinas Virtuais (VMs): Instâncias virtualizadas que simulam um sistema completo, com sistema operacional e aplicações.
    • Recursos Físicos: Hardware sobre o qual a virtualização é implementada, incluindo CPU, memória, armazenamento e rede.

    2. Estrutura Completa do Tema (Nível Prova)#

    Regras#

    • Isolamento: Cada VM deve operar de forma independente, sem afetar as demais VMs.
    • Encapsulamento: O estado completo de uma VM é armazenado em arquivos, facilitando migrações e backups.
    • Compatibilidade: VMs devem ser capazes de rodar qualquer sistema operacional e aplicação, independente do hardware subjacente.

    Classificações#

    • Tipo 1 (Bare Metal): Hypervisors que rodam diretamente no hardware, como VMware ESXi e Microsoft Hyper-V.
    • Tipo 2 (Hosted): Hypervisors que rodam sobre um sistema operacional existente, como VMware Workstation e Oracle VirtualBox.

    Categorias#

    • Virtualização de Servidor: Focada em dividir um servidor físico em múltiplos servidores virtuais.
    • Virtualização de Rede: Cria redes virtuais independentes em uma única rede física.
    • Virtualização de Armazenamento: Permite gerenciar o armazenamento físico de forma flexível e eficiente.

    Exceções#

    • Paravirtualização: Uma VM é ciente de sua natureza virtualizada e pode comunicar-se diretamente com o hypervisor para otimizar o desempenho.

    3. Regras Técnicas e Comportamento#

    • Overcommitment: Alocar mais recursos virtuais do que a capacidade física real, confiando que nem todas as VMs utilizarão seus recursos ao máximo simultaneamente.
    • Snapshotting: Capacidade de salvar o estado de uma VM em um ponto no tempo, permitindo restaurações rápidas.

    4. Diferenças Críticas#

    • VMware vs. Hyper-V: VMware oferece maior escalabilidade e recursos avançados como vMotion, enquanto Hyper-V é integrado ao Windows Server, podendo ser uma opção mais econômica para ambientes Windows.
    • Tipo 1 vs. Tipo 2: Hypervisors do Tipo 1 oferecem melhor desempenho e segurança por rodarem diretamente no hardware, enquanto os do Tipo 2 são mais fáceis de instalar e gerenciar, sendo ideais para testes e ambientes de desenvolvimento.

    5. Exemplos e Aplicações#

    • Consolidação de Servidores: Reduzir o número de servidores físicos, executando múltiplos servidores virtuais em um único hardware.
    • Ambientes de Teste e Desenvolvimento: Criar e descartar rapidamente ambientes isolados para testar novas aplicações ou atualizações.

    6. Pegadinhas de Prova#

    • Virtualização vs. Emulação: A virtualização permite executar sistemas operacionais diretamente no hardware real (com o auxílio do hypervisor), enquanto a emulação simula todo o hardware via software, sendo significativamente mais lenta.
    • Virtualização Completa vs. Paravirtualização: A virtualização completa não requer modificações no sistema operacional convidado, enquanto a paravirtualização exige.

    8. Simulado (Certo ou Errado)#

    1. ( ) Hypervisors do Tipo 1 são instalados sobre um sistema operacional existente.
    2. ( ) A virtualização de rede permite a criação de múltiplas redes virtuais isoladas em uma única infraestrutura física.
    3. ( ) VMware ESXi é um exemplo de hypervisor do Tipo 2.
    4. ( ) A paravirtualização exige que o sistema operacional convidado seja modificado para melhorar o desempenho.
    5. ( ) Overcommitment é uma prática segura e recomendada sem qualquer planejamento prévio.
    6. ( ) Snapshots permitem restaurar uma VM para um estado anterior, facilitando o gerenciamento de versões e backups.
    7. ( ) Virtualização e emulação são termos intercambiáveis e referem-se ao mesmo processo tecnológico.
    8. ( ) Hyper-V pode ser uma escolha econômica para ambientes predominantemente Windows, oferecendo integração e gestão simplificadas.
