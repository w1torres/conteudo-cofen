    2. Modelo Entidade-Relacionamento#

    1. Fundamentos e Funcionamento#

    O Modelo Entidade-Relacionamento (MER) é uma abordagem de modelagem de dados que visa representar as entidades relevantes para um domínio de negócio e as relações entre essas entidades. Este modelo é amplamente utilizado na fase de modelagem conceitual de sistemas de banco de dados. Seu funcionamento interno baseia-se na abstração de entidades do mundo real em um esquema que pode ser facilmente traduzido para um modelo de dados relacional.

    Componentes principais:#

    • Entidades: Objetos ou conceitos que possuem existência independente. Exemplo: Cliente, Pedido.
    • Relacionamentos: Associações entre duas ou mais entidades. Exemplo: Um cliente faz um pedido.
    • Atributos: Características ou propriedades das entidades ou relacionamentos. Exemplo: Nome do Cliente, Data do Pedido.

    2. Estrutura Completa do Tema (Nível Prova)#

    Elementos relevantes:#

    • Entidades Fortes e Fracas: Entidades fortes existem independentemente, enquanto entidades fracas dependem de outra entidade para existir.
    • Relacionamentos de Cardinalidade: Define quantas instâncias de uma entidade estão associadas a outra. Exemplos: 1:1, 1:N, N:N.
    • Atributos Simples, Compostos e Multivalorados: Simples são indivisíveis, compostos podem ser divididos em partes menores, e multivalorados podem ter múltiplos valores para a mesma entidade.
    • Atributos Derivados: Atributos cujos valores são derivados de outros atributos.
    • Chaves Primárias e Estrangeiras: Identificam unicamente as instâncias das entidades e estabelecem relações entre elas.

    3. Regras Técnicas e Comportamento#

    • Entidades devem ser nomeadas no singular e com substantivos.
    • Relacionamentos devem expressar ações ou associações e geralmente são nomeados com verbos.
    • Atributos devem ser atômicos no caso de modelagem conceitual.
    • Chaves Primárias não podem ser nulas e devem ser únicas para cada instância da entidade.
    • Chaves Estrangeiras são usadas para estabelecer relações e podem ser repetidas nas entidades que representam o lado N em uma relação 1:N.

    4. Diferenças Críticas#

    • MER vs Modelo Relacional: O MER é usado na modelagem conceitual, focando na estrutura e significado dos dados, enquanto o Modelo Relacional é usado na modelagem lógica, focando na organização dos dados em tabelas.
    • Entidades Fortes vs Fracas: Entidades fortes podem existir por si só, enquanto entidades fracas dependem de uma entidade forte para sua existência.

    5. Exemplos e Aplicações#

    • Diagrama E-R: Representação gráfica do modelo, onde entidades são representadas por retângulos, relacionamentos por losangos e atributos por círculos.
    • Cenário de Vendas: Entidade Cliente (com atributos como Nome e ID_Cliente), entidade Produto (com atributos como Nome e ID_Produto), e o relacionamento Faz entre Cliente e Produto.

    6. Pegadinhas de Prova#

    • Afirmar que atributos compostos não podem ser parte de entidades.
    • Dizer que uma chave estrangeira sempre corresponde a uma chave primária na mesma tabela.
    • Confundir relacionamentos N:N como sendo representados diretamente sem uma entidade associativa.

    7. Tabela de Fixação#

    8. Simulado (Certo ou Errado)#

    1. Entidades fracas sempre precisam de uma entidade forte para existir. (Certo)
    2. Atributos derivados são calculados a partir de outros atributos e não precisam ser armazenados fisicamente. (Certo)
    3. Relacionamentos N:N podem ser representados diretamente no modelo relacional sem a necessidade de uma entidade associativa. (Errado)
    4. Chaves estrangeiras são utilizadas para garantir a integridade referencial entre entidades. (Certo)
    5. Atributos compostos podem ser divididos em partes menores, mas não podem ser incluídos em um modelo conceitual. (Errado)
    6. Uma chave primária pode ser composta por mais de um atributo para garantir a unicidade da entidade. (Certo)
    7. Todos os atributos de uma entidade devem ser atômicos no modelo relacional. (Certo)
    8. Relacionamentos 1:1 indicam que uma instância de uma entidade está associada a no máximo uma instância de outra entidade. (Certo)

    Gabarito comentado:#

    1. Certo: Entidades fracas dependem de uma entidade forte para sua definição e existência.
    2. Certo: Atributos derivados são baseados em outros atributos e podem ser calculados quando necessário, não sendo obrigatória sua armazenagem.
    3. Errado: Relacionamentos N:N no modelo relacional requerem uma entidade associativa para sua correta representação.
    4. Certo: Chaves estrangeiras são fundamentais para manter a integridade referencial, vinculando registros de diferentes tabelas.
    5. Errado: Atributos compostos são perfeitamente aceitáveis em um modelo conceitual, ajudando a detalhar a estrutura dos dados.
    6. Certo: Chaves primárias compostas são usadas quando um único atributo não é suficiente para garantir a unicidade.
    7. Certo: No modelo relacional, é uma prática comum que todos os atributos sejam atômicos, facilitando a manipulação dos dados.
    8. Certo: Relacionamentos 1:1 refletem uma associação direta e exclusiva entre duas instâncias de entidades.
