    3. Restrições de integridade#

    1. Fundamentos e Funcionamento#

    As restrições de integridade em bancos de dados são regras aplicadas para manter a precisão e a confiabilidade dos dados dentro de uma base de dados. Elas garantem que as informações inseridas, atualizadas ou deletadas respeitem a lógica do domínio de aplicação e as regras de negócio definidas. As restrições de integridade são implementadas através de diferentes componentes, como chaves primárias, chaves estrangeiras, restrições de domínio (como tipos de dados e tamanhos), e restrições de integridade referencial.

    2. Estrutura Completa do Tema (Nível Prova)#

    • Regras:

      • Chaves Primárias (PK): Identificam unicamente cada registro na tabela.
      • Chaves Estrangeiras (FK): Estabelecem uma ligação entre tabelas, garantindo a integridade referencial.
      • Restrições de Domínio: Definem os tipos de dados permitidos, seus tamanhos e formatos.
      • Restrições de Integridade Referencial: Garantem que relações entre tabelas sejam preservadas.
      • Restrições Únicas: Asseguram que todos os valores em uma coluna sejam únicos.
      • Restrições de Verificação (CHECK): Validam que os dados inseridos atendam a uma condição específica.

    • Classificações e Categorias:

      • Estáticas vs. Dinâmicas: Estáticas são verificadas no momento da inserção ou atualização. Dinâmicas são verificadas em tempo de consulta.
      • Declarativas vs. Procedurais: Declarativas são expressas diretamente na DDL. Procedurais são implementadas via triggers ou procedimentos armazenados.

    • Exceções:

      • Desabilitação temporária de restrições para carga de dados em massa.
      • Uso de restrições NOCHECK para manter dados históricos que não atendem às restrições atuais.

    3. Regras Técnicas e Comportamento#

    • Chaves Primárias (PK): Não podem conter valores nulos e devem ser únicas na tabela.
    • Chaves Estrangeiras (FK): Devem corresponder a uma chave primária existente ou a uma chave única na tabela referenciada.
    • Restrições de Domínio: Limitam os tipos de dados e seus valores possíveis (por exemplo, ENUM).
    • Restrições de Integridade Referencial: Impedem a exclusão de dados que estão sendo referenciados por outras tabelas.
    • Restrições Únicas: Podem ser aplicadas a uma ou mais colunas.
    • Restrições de Verificação (CHECK): Podem incluir expressões lógicas para validar dados.

    4. Diferenças Críticas#

    • Chave Primária vs. Chave Estrangeira: A chave primária identifica unicamente registros dentro de sua própria tabela, enquanto a chave estrangeira é uma referência a uma chave primária ou única em outra tabela, usada para manter a integridade referencial.
    • Restrições Únicas vs. Chaves Primárias: Ambas garantem unicidade, mas uma tabela pode ter várias restrições únicas e apenas uma chave primária.

    5. Exemplos e Aplicações#

    CREATE TABLE Cliente (
    ClienteID INT PRIMARY KEY,
    Nome VARCHAR(100),
    Email VARCHAR(100) UNIQUE,
    DataNascimento DATE CHECK (DataNascimento > '1900-01-01')
);

CREATE TABLE Pedido (
    PedidoID INT PRIMARY KEY,
    ClienteID INT FOREIGN KEY REFERENCES Cliente(ClienteID),
    DataPedido DATE,
    Total DECIMAL(10, 2)
);

    6. Pegadinhas de Prova#

    • Afirmar que chaves estrangeiras podem referenciar qualquer coluna, mesmo que não seja uma chave primária ou única.
    • Dizer que restrições de integridade referencial apenas impedem a inserção de dados inválidos, omitindo que também afetam a exclusão e atualização.
    • Confundir restrições únicas com chaves primárias, sugerindo que uma tabela pode ter múltiplas chaves primárias.

    8. Simulado (Certo ou Errado)#

    1. Uma chave estrangeira pode referenciar qualquer coluna de outra tabela. (Errado)
    2. Restrições de integridade referencial são aplicadas apenas no momento da inserção de dados. (Errado)
    3. Uma tabela pode ter várias chaves primárias. (Errado)
    4. Restrições de domínio incluem a definição de tipos de dados e tamanhos de campos. (Certo)
    5. Restrições de verificação (CHECK) podem incluir expressões lógicas para validar os dados inseridos. (Certo)
    6. Chaves estrangeiras ajudam a manter a integridade referencial entre tabelas. (Certo)
    7. Restrições únicas e chaves primárias são funcionalmente idênticas. (Errado)
    8. Uma restrição de integridade referencial pode impedir a exclusão de um registro se ele estiver sendo referenciado por outra tabela. (Certo)

    Gabarito comentado:#

    1. Errado. Uma chave estrangeira deve referenciar uma chave primária ou uma chave única na tabela referenciada.
    2. Errado. Restrições de integridade referencial são aplicadas em inserções, atualizações e exclusões para garantir a integridade dos dados.
    3. Errado. Uma tabela só pode ter uma chave primária, mas pode ter múltiplas restrições únicas.
    4. Certo. Restrições de domínio definem os tipos de dados permitidos, seus tamanhos e formatos.
    5. Certo. Restrições de verificação (CHECK) são usadas para garantir que os dados atendam a condições específicas.
    6. Certo. Chaves estrangeiras são fundamentais para manter a integridade referencial, assegurando que as relações entre tabelas sejam consistentes.
    7. Errado. Apesar de ambas garantirem unicidade, restrições únicas podem ser aplicadas a uma ou mais colunas, enquanto uma chave primária é única por tabela.
    8. Certo. Isso é um exemplo de restrição de integridade referencial, impedindo ações que violariam a integridade dos dados entre tabelas relacionadas.
