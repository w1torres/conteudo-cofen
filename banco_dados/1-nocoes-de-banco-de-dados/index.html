    1. Noções de banco de dados#

    1. Fundamentos e Funcionamento#

    • Definição técnica: Um banco de dados é um conjunto organizado de dados, geralmente armazenados e acessíveis eletronicamente a partir de um sistema de computador. Os bancos de dados são mais frequentemente organizados para modelar aspectos da realidade de uma maneira que suporta processos que requerem informações, como modelagem de dados, análise de dados, armazenamento e recuperação de informações.
    • Como funciona internamente: Internamente, um banco de dados usa um motor de banco de dados que é responsável por armazenar, recuperar, atualizar e deletar dados. Este motor opera sobre um modelo de dados, como o relacional, que define a estrutura lógica do banco de dados e permite a manipulação de dados através de uma linguagem de consulta, como SQL.
    • Componentes principais: Os componentes principais de um banco de dados incluem tabelas que armazenam registros de dados, campos que armazenam valores de dados, chaves primárias que identificam unicamente cada registro, e relações que definem como os dados em diferentes tabelas estão conectados.

    2. Estrutura Completa do Tema (Nível Prova)#

    • Regras: Normalização, ACID (Atomicidade, Consistência, Isolamento, Durabilidade)
    • Classificações: Relacional, Não Relacional (NoSQL), Distribuído, Orientado a Objetos
    • Categorias: Operacionais, Analíticos, Data Warehouses, Data Marts
    • Exceções: Sistemas de arquivos não são considerados bancos de dados, embora possam armazenar dados.

    3. Regras Técnicas e Comportamento#

    • Funcionamento detalhado: A normalização é um processo de organização de dados em um banco de dados que inclui a criação de tabelas e a definição de relações com o objetivo de reduzir a redundância e melhorar a integridade dos dados. As propriedades ACID garantem transações confiáveis e seguras.
    • Limitações: Bancos de dados relacionais podem não escalar horizontalmente tão bem quanto os NoSQL em ambientes distribuídos.
    • Condições de uso: Escolha entre relacional e NoSQL baseia-se em requisitos como consistência, disponibilidade, e tolerância a partição (CAP theorem).

    4. Diferenças Críticas#

    • Comparar com conceitos semelhantes: Banco de dados vs. Sistema de Gerenciamento de Banco de Dados (SGBD) - O banco de dados é o conjunto de dados propriamente dito, enquanto o SGBD é o software que interage com o usuário, aplicações, e o banco de dados.
    • Destacar o que a banca confunde: A banca pode confundir normalização com modelagem de dados. Normalização é um subconjunto da modelagem de dados focado na organização dos dados.

    5. Exemplos e Aplicações#

    • Exemplos técnicos (código se aplicável):
    CREATE TABLE Cliente (
    ClienteID int PRIMARY KEY,
    Nome varchar(255),
    Email varchar(255)
);
    • Cenários reais: Um sistema de e-commerce pode usar um banco de dados relacional para armazenar informações sobre clientes, pedidos, e produtos, enquanto um sistema de análise de grandes volumes de dados pode preferir um banco de dados NoSQL para melhor desempenho e escalabilidade.

    6. Pegadinhas de Prova#

    • Erros comuns: Confundir transação com consulta.
    • Afirmações que parecem corretas: “Bancos de dados NoSQL não suportam transações.” - Isso é falso, pois alguns sistemas NoSQL suportam transações.
    • Termos que invalidam questões: “Bancos de dados relacionais não podem escalar.” - Eles podem escalar verticalmente e, com técnicas específicas, também horizontalmente.

    8. Simulado (Certo ou Errado)#

    1. ( ) Bancos de dados e SGBDs são termos intercambiáveis.
    2. ( ) A normalização é um processo exclusivo de bancos de dados relacionais.
    3. ( ) Propriedades ACID não são relevantes para bancos de dados NoSQL.
    4. ( ) Bancos de dados NoSQL são inerentemente não relacionais.
    5. ( ) Uma chave primária pode identificar unicamente cada registro em uma tabela.
    6. ( ) Bancos de dados relacionais não podem ser usados para aplicações de big data.
    7. ( ) A escolha entre um banco de dados relacional e NoSQL deve considerar o teorema CAP.
    8. ( ) Todas as transações em um banco de dados devem seguir as propriedades ACID.

    Gabarito comentado:

    1. Errado. Bancos de dados referem-se ao conjunto de dados armazenados, enquanto SGBDs são os sistemas utilizados para gerenciar e acessar esses dados.
    2. Errado. Embora frequentemente associada a bancos de dados relacionais, a normalização é um conceito que pode ser aplicado em diferentes contextos de organização de dados.
    3. Errado. Propriedades ACID são relevantes para garantir a confiabilidade das transações, independentemente do tipo de banco de dados.
    4. Errado. Embora muitos bancos de dados NoSQL sejam não relacionais, o termo NoSQL abrange uma variedade de tipos de bancos de dados, incluindo alguns que suportam relações entre dados.
    5. Certo. A chave primária é um identificador único para cada registro em uma tabela.
    6. Errado. Bancos de dados relacionais podem ser usados em aplicações de big data, especialmente quando combinados com técnicas de sharding ou particionamento.
    7. Certo. O teorema CAP (Consistência, Disponibilidade, Tolerância a Partição) é uma consideração importante na escolha entre bancos de dados relacionais e NoSQL.
    8. Errado. Enquanto as propriedades ACID são importantes, nem todas as transações necessariamente precisam seguir estritamente essas propriedades, dependendo do contexto e dos requisitos específicos da aplicação.
