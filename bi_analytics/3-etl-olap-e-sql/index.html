    3. ETL, OLAP e SQL#

    1. Fundamentos e Funcionamento#

    Definição técnica#

    • ETL (Extract, Transform, Load): Processo de extração de dados de diversas fontes, transformação desses dados conforme regras de negócio e carga dos dados transformados em um sistema de destino, geralmente um Data Warehouse.
    • OLAP (Online Analytical Processing): Categoria de software que permite aos usuários analisar informações de várias fontes de banco de dados de maneira multidimensional.
    • SQL (Structured Query Language): Linguagem padrão para gerenciamento de banco de dados, usada para consulta, inserção, atualização e modificação de dados.

    Como funciona internamente#

    • ETL:
      • Extract: Os dados são extraídos de sistemas de origem.
      • Transform: Os dados extraídos são limpos, mapeados e transformados.
      • Load: Os dados transformados são carregados em um repositório de destino.
    • OLAP: Utiliza cubos OLAP, que são estruturas de dados multidimensionais que permitem rápida análise de dados.
    • SQL: Opera através de comandos que permitem a manipulação e consulta de dados em bancos de dados relacionais.

    Componentes principais#

    • ETL: Ferramentas de ETL, área de staging, regras de transformação.
    • OLAP: Cubos OLAP, servidores OLAP, ferramentas de consulta OLAP.
    • SQL: Banco de dados, tabelas, índices, views, procedimentos armazenados.

    2. Estrutura Completa do Tema (Nível Prova)#

    • Regras:
      • ETL: Sequencialidade das etapas, integridade dos dados.
      • OLAP: Multidimensionalidade, agregação e drill-down/up.
      • SQL: Sintaxe, operadores, cláusulas.
    • Classificações:
      • ETL: Batch vs. Real-time.
      • OLAP: ROLAP, MOLAP, HOLAP.
      • SQL: DDL, DML, DCL, TCL.
    • Categorias:
      • ETL: Ferramentas (Informatica, Talend, SSIS).
      • OLAP: Ferramentas (Microsoft Analysis Services, Oracle OLAP).
      • SQL: Bancos de dados (MySQL, Oracle, SQL Server).
    • Exceções:
      • ETL: Exceções de transformação, erros de carga.
      • OLAP: Limitações de dados não estruturados.
      • SQL: Limitações de subconsultas, restrições de índices.

    3. Regras Técnicas e Comportamento#

    • ETL: A transformação deve ser idempotente, garantindo que, independentemente de quantas vezes seja executada, o resultado será sempre o mesmo.
    • OLAP: A análise deve permitir a visualização dos dados em diferentes níveis de agregação sem alterar os dados subjacentes.
    • SQL: Deve-se usar índices para melhorar a performance de consultas, mas com a consciência de que eles podem desacelerar operações de inserção e atualização.

    4. Diferenças Críticas#

    • ETL vs. ELT: O ETL processa os dados antes de carregá-los no Data Warehouse, enquanto o ELT carrega os dados primeiro e depois os processa.
    • OLAP vs. OLTP: OLAP é usado para análise e processamento de grandes volumes de dados, enquanto OLTP é otimizado para transações diárias e operações.
    • SQL vs. NoSQL: SQL é usado em bancos de dados relacionais com esquemas fixos, enquanto NoSQL é usado em bancos de dados não relacionais, suportando esquemas dinâmicos.

    5. Exemplos e Aplicações#

    ETL#

    -- Exemplo de transformação SQL em uma etapa de ETL
SELECT nome, UPPER(sobrenome) as sobrenome_maiusculo
FROM clientes;

    OLAP#

    • Cenário real: Análise de vendas por região, produto e tempo, permitindo que os gestores vejam não apenas o total de vendas, mas também como as vendas se comparam entre diferentes regiões ou períodos.

    SQL#

    -- Exemplo de consulta SQL
SELECT nome, email
FROM clientes
WHERE cidade = 'São Paulo';

    6. Pegadinhas de Prova#

    • Afirmar que ETL não permite transformações complexas.
    • Confundir OLAP com OLTP em cenários de análise de dados.
    • Dizer que SQL não pode ser usado para manipulação de dados não relacionais.

    8. Simulado (Certo ou Errado)#

    1. O processo de ETL é fundamentalmente sequencial e não pode ser executado em paralelo. (ERRADO)
    2. OLAP permite a análise de dados em várias dimensões simultaneamente. (CERTO)
    3. SQL é exclusivamente utilizado para bancos de dados relacionais. (ERRADO)
    4. A transformação em ETL deve ser projetada para ser idempotente. (CERTO)
    5. OLTP é mais adequado para análise de dados do que OLAP. (ERRADO)
    6. ETL pode ser substituído completamente por ELT em todos os cenários. (ERRADO)
    7. NoSQL é uma alternativa que elimina a necessidade de SQL. (ERRADO)
    8. MOLAP, ROLAP e HOLAP são classificações de OLAP baseadas no tipo de armazenamento. (CERTO)

    Gabarito comentado:#

    1. O ETL pode ser paralelizado em determinadas fases, especialmente durante a carga e transformação, dependendo da ferramenta utilizada.
    2. OLAP é projetado especificamente para análise multidimensional, permitindo aos usuários visualizar dados sob várias perspectivas.
    3. SQL, embora predominantemente usado em bancos de dados relacionais, também pode ser aplicado em alguns sistemas NoSQL que suportam SQL-like queries.
    4. A idempotência na transformação garante que os dados possam ser processados múltiplas vezes sem duplicação ou perda de informação.
    5. OLTP é otimizado para transações e operações do dia a dia, enquanto OLAP é especializado na análise e consulta de grandes volumes de dados.
    6. ELT é uma abordagem alternativa ao ETL, mas cada um tem seus próprios casos de uso e vantagens dependendo dos requisitos específicos do projeto.
    7. NoSQL oferece alternativas para casos de uso onde esquemas flexíveis ou armazenamento de dados não estruturados são necessários, mas não elimina a necessidade de SQL em todos os cenários.
    8. MOLAP, ROLAP e HOLAP diferem na maneira como os dados são armazenados e processados, com MOLAP usando armazenamento multidimensional, ROLAP usando bancos de dados relacionais, e HOLAP uma combinação dos dois.
