    2. Arquitetura analítica#

    1. Fundamentos e Funcionamento#

    A arquitetura analítica refere-se à estruturação de tecnologias e processos que permitem a coleta, armazenamento, análise e visualização de dados para suportar a tomada de decisões baseada em dados. Internamente, ela é composta por várias camadas, incluindo a ingestão de dados, armazenamento de dados, processamento/análise de dados, e a camada de apresentação ou visualização.

    2. Estrutura Completa do Tema (Nível Prova)#

    • Ingestão de Dados: Inclui mecanismos para coletar dados de diversas fontes.
    • Armazenamento de Dados: Utiliza tecnologias como Data Warehouses, Data Lakes, e bases de dados NoSQL.
    • Processamento e Análise de Dados: Emprega ferramentas de ETL/ELT, processamento em batch e em tempo real, e motores de análise.
    • Visualização de Dados e BI: Ferramentas de Business Intelligence (BI) e dashboards para apresentação dos insights.
    • Governança de Dados: Políticas e procedimentos para gerenciar a acessibilidade, integridade, segurança e qualidade dos dados.
    • Segurança de Dados: Inclui criptografia, autenticação, e autorização para proteger os dados.

    3. Regras Técnicas e Comportamento#

    • Ingestão de Dados: Deve ser capaz de lidar com dados estruturados e não estruturados de diversas fontes.
    • Armazenamento de Dados: Deve suportar a escalabilidade e a flexibilidade para armazenar grandes volumes de dados.
    • Processamento de Dados: Deve fornecer capacidades para processamento tanto em batch quanto em tempo real.
    • Visualização de Dados: Deve oferecer interatividade e a capacidade de gerar insights acionáveis.

    4. Diferenças Críticas#

    • Data Warehouse vs Data Lake: Data Warehouse é estruturado e otimizado para análise de dados, enquanto Data Lake pode armazenar dados brutos não estruturados.
    • ETL vs ELT: ETL (Extract, Transform, Load) transforma os dados antes de carregá-los no armazenamento de dados, enquanto ELT (Extract, Load, Transform) carrega os dados primeiro e depois os transforma.

    5. Exemplos e Aplicações#

    • ETL: Um processo ETL pode extrair dados de sistemas de vendas, transformar esses dados para um formato padronizado, e carregar no Data Warehouse para análise.
    • Data Lake: Uma empresa pode usar um Data Lake para armazenar todos os seus dados brutos, incluindo logs de web, transações financeiras, e mídias sociais para análise futura.

    6. Pegadinhas de Prova#

    • Afirmação: “Data Lakes substituem a necessidade de Data Warehouses.” (Errado, pois servem a propósitos diferentes).
    • Termo: “Processamento de dados em tempo real é menos importante que o batch.” (Errado, a importância varia de acordo com o caso de uso).

    7. Tabela de Fixação#

    ConceitoRegraPegadinha
    Data WarehouseEstruturado para análise rápida de dados.Confundir com Data Lake.
    Data LakeArmazena dados brutos não estruturados.Achar que é apenas para grandes volumes de dados.
    ETLTransforma os dados antes de carregá-los.Confundir a ordem das etapas com ELT.
    ELTCarrega os dados antes de transformá-los.Pensar que é inferior ao ETL.
    Processamento BatchProcessa grandes volumes de dados de uma vez.Confundir com processamento em tempo real.
    Processamento Real-timeProcessa dados conforme são recebidos.Achar que não é viável para análises complexas.

    8. Simulado (Certo ou Errado)#

    1. Data Lakes são estruturados e otimizados para análise rápida de dados. (Errado)
    2. O processo ELT transforma os dados antes de carregá-los no sistema de armazenamento. (Errado)
    3. A ingestão de dados pode lidar tanto com dados estruturados quanto não estruturados. (Certo)
    4. Governança de dados não é considerada uma parte essencial da arquitetura analítica. (Errado)
    5. Processamento em tempo real é menos relevante para análises que não são sensíveis ao tempo. (Certo)
    6. Data Warehouses e Data Lakes podem coexistir dentro da mesma arquitetura analítica. (Certo)
    7. A visualização de dados não é necessária para a interpretação de insights gerados por análises. (Errado)
    8. Segurança de dados inclui apenas a criptografia de dados em repouso. (Errado)

    Gabarito comentado:#

    1. Errado. Data Lakes armazenam dados brutos não estruturados, permitindo armazenamento flexível, não otimizado especificamente para análise rápida.
    2. Errado. ELT carrega os dados primeiro e depois os transforma, ao contrário do ETL.
    3. Certo. A ingestão de dados é projetada para coletar dados de diversas fontes, incluindo estruturadas e não estruturadas.
    4. Errado. A governança de dados é crucial para garantir a qualidade, segurança, e acessibilidade dos dados dentro de uma organização.
    5. Certo. Processamento em tempo real é mais relevante para casos de uso onde a atualização imediata dos dados é crítica.
    6. Certo. Data Warehouses e Data Lakes servem a propósitos diferentes e podem ser usados complementarmente em uma arquitetura analítica.
    7. Errado. A visualização de dados é fundamental para transformar dados brutos em insights compreensíveis e acionáveis.
    8. Errado. Segurança de dados abrange uma variedade de práticas, incluindo criptografia de dados em trânsito e em repouso, autenticação, e autorização.
