    5. ETL e ELT#

    1. Fundamentos e Funcionamento#

    Definição técnica#

    ETL (Extract, Transform, Load) e ELT (Extract, Load, Transform) são processos utilizados no tratamento de dados, especialmente em ambientes de Data Warehousing. O ETL é o processo de extrair dados de diversas fontes, transformá-los conforme necessário (limpeza, agregação, etc.) e carregá-los em um destino, como um Data Warehouse. O ELT, por sua vez, modifica a ordem dessas operações, carregando os dados antes de transformá-los, o que é viável principalmente em ambientes de cloud computing com capacidades de processamento distribuído.

    Como funciona internamente#

    • ETL:

      1. Extração: Dados são extraídos de fontes heterogêneas.
      2. Transformação: Os dados extraídos são limpos, mapeados e transformados.
      3. Carga: Os dados transformados são carregados no sistema de destino.

    • ELT:

      1. Extração: Similar ao ETL, dados são extraídos de fontes diversas.
      2. Carga: Dados são carregados diretamente no destino.
      3. Transformação: As transformações são realizadas no próprio ambiente de destino.

    Componentes principais#

    • Fontes de Dados: Origem dos dados (bancos de dados, arquivos, APIs).
    • Área de Staging: Área intermediária para dados extraídos.
    • Ferramentas de Transformação: Softwares ou serviços que aplicam regras de negócio.
    • Destino: Data Warehouse, Data Lake ou qualquer sistema de armazenamento.

    2. Estrutura Completa do Tema (Nível Prova)#

    • Regras:

      • A ordem das operações em ETL é estritamente seguida.
      • Em ELT, a transformação ocorre após a carga, aproveitando a capacidade de processamento do destino.

    • Classificações:

      • ETL Tradicional: Para ambientes onde o processamento é feito antes da carga.
      • ETL em Tempo Real: Processamento e carga de dados em tempo real.
      • ELT para Big Data: Utilizado em ambientes de big data, onde a transformação é distribuída.

    • Categorias:

      • Ferramentas de ETL/ELT: Informatica PowerCenter, Talend, Apache Nifi.
      • Ambientes de Destino: Data Warehouses (Snowflake, Redshift), Data Lakes.

    • Exceções:

      • Casos de uso onde a transformação não é necessária antes da carga.

    3. Regras Técnicas e Comportamento#

    • Funcionamento detalhado:

      • ETL requer uma área de staging robusta para transformações.
      • ELT aproveita a escalabilidade e poder de processamento dos sistemas de destino modernos.

    • Limitações:

      • ETL pode ser mais lento devido ao processamento intermediário.
      • ELT requer destinos com capacidades de processamento avançadas.

    • Condições de uso:

      • ETL é preferível quando as transformações são complexas e o volume de dados é gerenciável.
      • ELT é ideal para grandes volumes de dados e quando o destino suporta processamento distribuído.

    4. Diferenças Críticas#

    • ETL é mais controlado e seguro para transformações complexas, enquanto ELT é mais rápido e escalável para grandes volumes de dados.
    • A banca pode confundir a necessidade de uma área de staging robusta em ELT, que na verdade é mais crítica em ETL.

    5. Exemplos e Aplicações#

    • ETL: Limpeza e agregação de dados de vendas de múltiplos sistemas antes de carregar em um DW para relatórios.
    • ELT: Carregamento de logs brutos em um Data Lake e usando Spark para transformações e análises.

    6. Pegadinhas de Prova#

    • Afirmações que sugerem ELT não requer transformação.
    • Dizer que ETL não é adequado para big data.
    • Confundir a ordem das operações em ETL e ELT.

    8. Simulado (Certo ou Errado)#

    1. ETL é mais indicado para processamento em tempo real devido à sua eficiência. (ERRADO)
    2. ELT aproveita a capacidade de processamento do sistema de destino para transformações. (CERTO)
    3. A área de staging é mais crítica em processos de ELT do que em ETL. (ERRADO)
    4. Ferramentas de ETL/ELT não são aplicáveis em ambientes de Data Lake. (ERRADO)
    5. Transformações em ELT ocorrem antes da carga dos dados no sistema de destino. (ERRADO)
    6. ETL é preferível quando as transformações de dados são complexas e detalhadas. (CERTO)
    7. ELT é menos escalável que ETL quando se trata de grandes volumes de dados. (ERRADO)
    8. Em ETL, a carga dos dados ocorre antes das transformações. (ERRADO)

    Gabarito comentado:#

    1. ERRADO. ETL não é necessariamente mais eficiente para tempo real; sistemas de ELT podem ser mais adequados devido à capacidade de processamento distribuído.
    2. CERTO. ELT realmente aproveita a capacidade de processamento do destino, permitindo transformações eficientes em grandes volumes de dados.
    3. ERRADO. A área de staging é uma componente crítica principalmente em processos de ETL, onde as transformações ocorrem antes da carga.
    4. ERRADO. Ferramentas de ETL/ELT são amplamente utilizadas em ambientes de Data Lake, especialmente para organizar e preparar dados para análise.
    5. ERRADO. Em ELT, a transformação ocorre após a carga dos dados, aproveitando o processamento do sistema de destino.
    6. CERTO. ETL é mais adequado para transformações complexas, pois oferece um controle mais fino sobre o processo de transformação.
    7. ERRADO. ELT é mais escalável que ETL em grandes volumes de dados, devido à sua capacidade de utilizar o processamento distribuído do destino.
    8. ERRADO. Em ETL, a transformação dos dados ocorre antes da carga, não o contrário.
